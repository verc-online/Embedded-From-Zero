# Embedded From Zero

> **Открытая книга по Embedded-разработке. От первого светодиода до реальных устройств.**

---

## Почему появился этот репозиторий

Большинство материалов по Embedded делятся на две крайности.

Первая — академические книги, которые подробно рассказывают про регистры и архитектуру микроконтроллеров, но редко показывают, как из этих знаний рождаются реальные устройства.

Вторая — бесконечные примеры на Arduino, где достаточно вызвать готовую функцию, не понимая, что происходит внутри.

Мне не хватало третьего пути.

Я захотел создать открытый репозиторий, который показывает весь путь обучения Embedded-разработке таким, каким он происходит на самом деле: с ошибками, поиском решений, изучением Datasheet, проектированием архитектуры и написанием собственных драйверов.

Этот репозиторий — не курс и не конспект.

Это инженерный дневник, который постепенно превращается в полноценную книгу.

---

# Цель проекта

Показать, что стать Embedded-разработчиком можно с нуля, если изучать не отдельные функции библиотек, а принципы работы микроконтроллеров и архитектуру программ.

Главная задача — научиться самостоятельно проектировать встроенные системы, понимать каждую строку собственного кода и не зависеть от готовых библиотек.

---

# Что уже реализовано

В рамках обучения уже были разработаны собственные драйверы и модули:

* GPIO
* Таймеры
* PWM
* UART
* I²C (TWI)
* EEPROM
* RTC DS3231
* LCD1602
* Scheduler
* Menu (FSM)
* Servo Driver

Все они используются в реальном проекте автоматической кормушки для животных.

# Как устроена каждая глава

Каждая тема проходит одинаковый путь:

1. Зачем это вообще нужно.
2. Как это работает внутри микроконтроллера.
3. Какие регистры участвуют.
4. Проектирование собственного драйвера.
5. Реализация.
6. Типичные ошибки.
7. Практическое применение.
8. Проверка себя.

Цель — не запомнить код, а научиться мыслить как Embedded-разработчик.

---

# Основные принципы

* Минимум магии.
* Максимум понимания.
* Никакого копирования без понимания.
* Каждый драйвер пишется самостоятельно.
* Архитектура важнее скорости.
* Datasheet — главный источник информации.

---

# Текущий проект

В качестве основного проекта используется автоматическая кормушка для домашних животных.

Проект включает:

* LCD-меню
* настройку времени
* расписание кормления
* управление сервоприводом
* сохранение настроек в EEPROM
* автоматическое кормление по времени

Именно на этом проекте постепенно изучаются все основные темы Embedded-разработки.

---

# Для кого этот репозиторий

Для тех, кто хочет изучать Embedded осознанно, понимать, что происходит внутри микроконтроллера, и научиться создавать собственные устройства, а не только использовать готовые библиотеки.

---

> **Не цель написать больше кода. Цель — стать инженером, который понимает каждое своё решение.**
