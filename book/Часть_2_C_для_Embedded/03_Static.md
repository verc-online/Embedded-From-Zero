# Static — память, которая не забывает

## История

Когда я только начинал изучать C, слово `static` казалось мне чем-то загадочным. В разных статьях его объясняли совершенно по-разному.

Где-то говорили, что `static` делает переменную постоянной.

Где-то — что он скрывает функции.

Где-то — что он хранит данные между вызовами.

Возникал закономерный вопрос: как одно ключевое слово может делать сразу столько разных вещей?

На самом деле никакой магии здесь нет. Просто `static` меняет своё поведение в зависимости от того, где он используется.

Именно поэтому он считается одним из самых непонятных модификаторов языка C.

В Embedded-разработке `static` используется практически в каждом драйвере. Если открыть исходники любой библиотеки или прошивки, можно увидеть десятки подобных строк:

```c
static uint8_t state;

static void Timer_Update(void);

static bool initialized;
```

Поэтому понять его работу необходимо как можно раньше.

---

# Постановка задачи

Представим простую ситуацию.

Есть функция, которая должна считать количество своих вызовов.

Наивная реализация выглядит так.

```c
void Counter(void)
{
    int count = 0;

    count++;

    printf("%d\n", count);
}
```

Кажется, всё правильно.

Но результат оказывается неожиданным.

```
1
1
1
1
```

Почему?

Потому что локальная переменная создаётся каждый раз заново.

При выходе из функции память освобождается.

При следующем вызове снова создаётся новая переменная со значением ноль.

Функция ничего не помнит.

Но что делать, если память всё-таки должна сохраниться?

---

# Простое объяснение

Представьте себе школьную доску.

Обычная локальная переменная похожа на запись мелом.

Написали число.

Посмотрели.

Стерли.

На следующем уроке доска снова пустая.

`static` работает иначе.

Это уже не запись мелом, а лист бумаги, который лежит в шкафу.

Каждый раз, когда вы приходите в кабинет, вы открываете шкаф и продолжаете писать на том же самом листе.

Поэтому функция помнит своё прошлое.

Именно это и делает `static` внутри функции.

```c
void Counter(void)
{
    static int count = 0;

    count++;

    printf("%d\n", count);
}
```

Теперь программа выведет

```
1
2
3
4
```

Переменная создаётся всего один раз и существует до завершения работы программы.

---

# Что происходит внутри микроконтроллера

Чтобы понять `static`, нужно вспомнить карту памяти микроконтроллера.

Упрощённо она выглядит так.

```text
Flash
│
├── Код программы
├── Константы
│
RAM
│
├── Data
├── BSS
├──────────────
│
├── Heap
│
├──────────────
│
└── Stack
```

Локальные переменные находятся в стеке.

Каждый вход в функцию создаёт новый участок памяти.

Каждый выход освобождает его.

Именно поэтому обычная переменная исчезает после выхода из функции.

Переменные `static` в стеке не живут.

Компилятор размещает их в области статических данных (`.data` или `.bss`).

Эта память выделяется ещё до входа в `main()`.

Поэтому значение остаётся доступным всё время работы программы.

---

# Две роли static

Большинство путаницы возникает потому, что `static` используется в двух совершенно разных ситуациях.

## 1. Static внутри функции

Здесь он изменяет время жизни переменной.

Без `static`

```c
void Foo(void)
{
    int counter = 0;

    counter++;
}
```

Переменная появляется при входе в функцию и исчезает при выходе.

Со `static`

```c
void Foo(void)
{
    static int counter = 0;

    counter++;
}
```

Переменная существует всё время работы программы.

---

## 2. Static на уровне файла

Здесь `static` уже не влияет на время жизни.

Он изменяет область видимости.

Например,

```c
static void Led_Update(void)
{
}
```

Такую функцию можно вызвать только из файла `led.c`.

Для остальных файлов проекта её как будто не существует.

То же самое относится и к переменным.

```c
static uint8_t ledState;
```

Теперь никакой другой модуль не сможет написать

```c
extern uint8_t ledState;
```

Компилятор просто не позволит этого сделать.

Таким образом модуль скрывает свои внутренние детали.

---

# Почему это важно в Embedded

Представим драйвер светодиода.

```text
led.c

Led_Init()
Led_On()
Led_Off()

Led_WriteRegister()
Led_UpdateHardware()
Led_CheckConfiguration()
```

Пользователь драйвера должен пользоваться только первыми тремя функциями.

Остальные являются внутренней реализацией.

Поэтому они объявляются так.

```c
static void Led_WriteRegister(void);

static void Led_UpdateHardware(void);

static void Led_CheckConfiguration(void);
```

Получается очень полезная архитектура.

```
Другие файлы проекта

        │

        ▼

Led_Init()
Led_On()
Led_Off()

──────────────

Led_WriteRegister()

Led_UpdateHardware()

Led_CheckConfiguration()
```

Внешний код не может случайно вызвать внутренние функции драйвера.

Это делает проект безопаснее и значительно упрощает сопровождение.

---

# Проектирование драйвера

Практически каждый драйвер содержит внутреннее состояние.

Например, драйвер кнопки.

```c
static bool previousState = false;
```

Каждый раз при вызове функции новое состояние сравнивается со старым.

Если убрать `static`, переменная будет каждый раз создаваться заново.

Получится примерно следующее.

```
false

↓

вызов функции

↓

false

↓

вызов функции

↓

false
```

Функция никогда не сможет узнать, какое состояние было раньше.

Следовательно, невозможно определить момент нажатия кнопки.

Именно поэтому практически во всех драйверах Embedded используются статические переменные.

---

# Разбор архитектуры

Во время написания драйверов полезно придерживаться простого правила.

Все внутренние функции делать `static`.

Например,

```c
static void Timer_Update(void);
static void Timer_Reset(void);
static bool Timer_CheckOverflow(void);
```

А наружу экспортировать только публичный интерфейс.

```c
void Timer_Init(void);
uint32_t Timer_GetTicks(void);
bool Timer_HasElapsed(...);
```

Такой подход делает модуль полностью самостоятельным.

Его внутреннюю реализацию можно изменить в любой момент, не затрагивая остальной проект.

# Типичные ошибки

### Ошибка №1

Считать, что `static` делает переменную константой.

Нет.

Она остаётся обычной переменной.

Её можно изменять сколько угодно.

---

### Ошибка №2

Думать, что статическая переменная создаётся при первом вызове функции.

Нет.

Память выделяется ещё до входа в `main()`.

---

### Ошибка №3

Забывать ставить `static` у внутренних функций драйвера.

В результате весь внутренний код становится доступен всему проекту.

Это нарушает инкапсуляцию и усложняет сопровождение.

---

### Ошибка №4

Путать две роли `static`.

Внутри функции он изменяет время жизни переменной.

На уровне файла — область видимости функций и переменных.

Это два разных механизма.

---

# Практическое применение

В Embedded `static` встречается буквально в каждом проекте.

Он используется:

* для хранения состояния конечных автоматов;
* для сохранения предыдущего состояния кнопок;
* для внутренних буферов драйверов;
* для счётчиков времени;
* для скрытия служебных функций;
* для защиты внутренней реализации модулей.

Со временем использование `static` становится привычкой.

Если функция нужна только внутри одного файла, её почти всегда объявляют `static`.

После изучения этой главы вы должны понимать:

* чем локальная переменная отличается от статической;

* зачем скрывать функции через `static`;

# Дополнение. Карта памяти микроконтроллера

До этого момента мы несколько раз говорили, что разные переменные хранятся в разных областях памяти. Теперь соберём всё вместе.

После включения питания память программы можно представить следующим образом.

```text
                FLASH
┌──────────────────────────────────────┐
│ Машинный код программы               │
│ Константы (const)                    │
│ Начальные значения static-переменных │
└──────────────────────────────────────┘


                 RAM
┌──────────────────────────────────────┐
│ .data                                │
│ Инициализированные глобальные        │
│ и static-переменные                  │
├──────────────────────────────────────┤
│ .bss                                 │
│ Неинициализированные глобальные      │
│ и static-переменные                  │
├──────────────────────────────────────┤
│                                      │
│ Heap (динамическая память)           │
│ растёт вверх                         │
│                                      │
├──────────────────────────────────────┤
│                                      │
│ свободная память                     │
│                                      │
├──────────────────────────────────────┤
│                                      │
│ Stack (локальные переменные)         │
│ растёт вниз                          │
│                                      │
└──────────────────────────────────────┘
```

---

## Flash

Flash — это энергонезависимая память.

Именно сюда компилятор помещает:

* машинный код программы;
* константы;
* начальные значения глобальных и статических переменных.

После выключения питания содержимое Flash сохраняется.

Именно поэтому микроконтроллер после включения снова запускает вашу прошивку.

---

## RAM

RAM — это оперативная память.

После подачи питания именно здесь появляются все рабочие данные программы.

При отключении питания её содержимое полностью теряется.

---

## Секция `.data`

Здесь располагаются все глобальные и `static`-переменные, которые имеют начальное значение.

Например,

```c
int counter = 10;

static bool initialized = true;
```

Во Flash компилятор хранит их первоначальные значения.

Во время запуска микроконтроллера специальный стартовый код копирует эти данные из Flash в RAM.

После этого программа начинает выполняться.

---

## Секция `.bss`

Если глобальная или `static`-переменная не имеет начального значения,

```c
int counter;

static bool initialized;
```

то компилятор помещает её в секцию `.bss`.

Перед запуском `main()` стартовый код просто заполняет эту область нулями.

Поэтому такие переменные автоматически становятся равны нулю.

---

## Stack

Стек используется для временных данных.

Именно здесь находятся:

* локальные переменные;
* параметры функций;
* адрес возврата после вызова функции.

При входе в функцию стек увеличивается.

При выходе память автоматически освобождается.

Поэтому локальные переменные не могут хранить данные между вызовами функций.

---

## Heap

Heap используется для динамического выделения памяти (`malloc()` и `free()`).

В большинстве Embedded-проектов его стараются вообще не использовать.

Причина проста: со временем память может фрагментироваться, а это делает поведение программы менее предсказуемым.

Именно поэтому во многих коммерческих проектах использование динамической памяти запрещено внутренними правилами разработки.

---

## Как всё это связано со `static`

Теперь становится понятно, почему `static` сохраняет своё значение.

Локальная переменная

```c
int value;
```

попадает в стек.

После выхода из функции стек освобождается, и переменная исчезает.

Статическая переменная

```c
static int value;
```

вообще не использует стек.

Она находится в секции `.data` или `.bss` и существует с момента запуска микроконтроллера до его выключения.

Именно поэтому она "помнит" своё предыдущее значение между вызовами функции.

---

> **Важно.** Мы ещё не раз будем возвращаться к этой схеме. Когда начнём изучать GPIO, таймеры, UART, I²C и SPI, станет понятно, что понимание карты памяти не менее важно, чем знание самого языка C. Embedded-разработчик постоянно работает с памятью микроконтроллера, поэтому эта схема постепенно станет для вас такой же привычной, как таблица умножения для школьника.
